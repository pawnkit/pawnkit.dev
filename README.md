# pawnkit.dev

This repository builds the public documentation site for PawnKit. It combines
short, task-based guides with reference material generated by the projects that
own it.

The site has no server-side component. Search runs in the browser, and schemas
are available from stable raw URLs.

## Build the site

Clone `pawnkit-spec`, `pawn-api`, and `pawnlint` beside this repository, then run:

```sh
go run ./cmd/site
```

The result is written to `dist`. Use any static file server to preview it:

```sh
go run github.com/caddyserver/caddy/v2/cmd/caddy@latest file-server --root dist
```

Run `go test ./...` before opening a pull request. Tests build the whole site,
check internal links, and verify source provenance and stable URLs.

## Where content belongs

Write practical guides in `content`. RFCs, schemas, API data, and lint rules stay
in their owning repositories and are listed in `sources.json` at a pinned tag or
commit.

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full workflow.

The [site policy](content/about-this-site.md) explains source ownership,
versioned URLs, and compatibility. Maintainers should also read the
[deployment notes](docs/deployment.md) before enabling GitHub Pages.
